Detailed Description

A threading class just like good old std::thread, but better.

This class is derived from std::jthread, which has a few improvements over std::thread: It is automatically joined in the destructor and it has the capability of being interrupted by another thread. What b::thread does additionally is to simply catch any unhandled exceptions in the root of the thread. This way you get clear error messages that something has failed, instead of your program randomly crashing.

Example when calling a class member function: b::thread(&MyClass::myMember, this)

static void b::thread::catch_common_exceptions ( Args &&...  args)
inlinestatic

Run a function and catch all exceptions with nice error messages.

Just specify a large lambda, containing everything to execute. When any exception is not handled and caught, an error is printed with an optional error message box. See b::thread::enable_message_box_on_exception() and b::thread::enable_catch_common_exceptions() to set this behaviour. If the message box is disabled, only the error message is printed to the console, if catch_common_exceptions is disabled, no exception is caught at all. Even though the main thread is not a b::thread, the main thread is also impacted by these settings. Although this function is used mainly internally, you could also use this function to create your own safe thread.

Defaults: In Debug mode, both settings are enabled, in Release mode only the message box is disabled.
