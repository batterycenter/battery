12 // Collision calculations done with the formula: e^((-k * (k - 1)/(2 * N)))
13 // The approximation error tends to zero as N increases, and we are dealing with a large N
14 // https://preshing.com/20110504/hash-collision-probabilities/
15 // With 10,000 registered types the probabilities of a collsion are:
16 // 64 bit hash: 2.71027645e-12 === 1 - 1/e^(6249375/2305843009213693952)
17 // 128 bit hash: 1.469221e-31 === 1 - 1/e^(6249375/42535295865117307932921825928971026432)
18 // 256 bit hash: 4.317652e-70 === 1 -
19 // 1/e^(6249375/14474011154664524427946373126085988481658748083205070504932198000989141204992) Probability of winning
20 // the Mega Millions lottery: 1 / 302,575,350 === 3.30496189e-9 Number of times winning to equal a collision chance: 64
21 // bit: 1220 times 128 bit: 2.2494655e22 times (or 22 sextillion times) 256 bit: 7.6545351e60 times From these
22 // calculations it is apparent that a 128 bit hash is more than sufficient
